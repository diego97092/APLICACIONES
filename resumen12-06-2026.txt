# MARKETPLACE HUB V2 – RESUMEN ACTUAL DEL PROYECTO

## OBJETIVO GENERAL

Crear un CRM propio para administrar múltiples cuentas de Facebook Marketplace desde una sola plataforma centralizada.

La arquitectura actual es:

Facebook Messenger Marketplace
↓
Extensión Chrome
↓
FastAPI
↓
SQLite
↓
CRM Marketplace Hub V2

Objetivo: manejar 25, 50 y hasta 100 perfiles sin Playwright, sin CDP, sin puertos debug, sin Telegram como interfaz principal y sin navegadores monitoreados de forma pesada.

# BASURA QUE DEBE FILTRARSE DEL HISTORIAL

No guardar como mensajes:

```text
Marketplace
View buyer
More options
View more items
No longer available
Facebook Marketplace Assistant
Meta AI
Con el objetivo de identificar y reducir las estafas...
Este mensaje se envió...
Messenger | Facebook
Enviar una respuesta rápida
Sí, ¿estás interesado?
Lo estoy mirando. Te avisaré.
Lo sentimos, no está disponible.
```

Guardar solo:

* Mensajes reales del cliente.
* Mensajes reales enviados por nosotros.
* Fecha/hora si aparece.
* Producto/precio solo como datos de conversación, no como mensajes repetidos.

# CONCLUSIÓN

El proyecto avanzó bastante.
La base ya existe: CRM + backend + SQLite + extensión.

Pero la clave absoluta del proyecto es la extensión.

Antes de avanzar a responder desde CRM, se debe dejar estable:

```text
Messenger → cajón Marketplace → bandeja → abrir cualquier chat desde CRM con scroll → capturar historial limpio
```

Cuando eso quede estable, ya se puede pasar a:

```text
CRM → escribir respuesta → extensión → Messenger → enviar mensaje
```
